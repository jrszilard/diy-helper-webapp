LLM Council Verdict

Unified Landing Page Chat Hub — Hero-to-Chat Morph Design Review
5 advisors | 5 peer reviews | April 1, 2026

The Question

Should we consolidate a DIY home improvement webapp from a landing page + separate /chat page into a single "Hero-to-Chat Morph" pattern? The landing page becomes the primary experience with unified AI intent classification replacing separate Quick Answer and Plan a Project tabs. All power features surface via header drawers. Key risk: users bouncing to ChatGPT.

Advisor Positions at a Glance

The Contrarian Intent router is untested load-bearing wall. Drawers bury core features. Morph hides differentiators at the wrong moment.
First Principles Wrong problem. Chat box = ChatGPT wrapper mental model. Must show irreplaceable value in 8 seconds before asking for input.
The Expansionist Morph is right but undersold. Lead with outputs not inputs. "Home brain" memory is the real moat. Chips should signal capabilities.
The Outsider Tab naming implies AI isn't authoritative. Headline doesn't answer "why not ChatGPT?" Morph transition is disorienting without signals.
The Executor Existing code supports this — 3-week build. Ship morph first, validate, then tackle wizard replacement.

Where the Council Clashes

Tension: Ship Fast vs. Rethink First
The Executor says ship the morph in 3 weeks with existing code. First Principles says the pattern leads with the wrong thing and no iteration will fix it. Resolution: Both can be true — ship the morph but redesign what the hero says and what the first response shows.
Tension: Plan CTA — Trap or Core Flow?
The Contrarian calls "Plan This Project" a commitment trap. The Executor treats it as straightforward. Resolution: The CTA needs to be earned — produce visible concrete output first, then the CTA is a natural next step.

Blind Spots the Council Caught

These emerged only through the peer review round — no individual advisor addressed them:

Trust failure and error states are existential. For a tool giving advice about electrical panels, structural loads, and building codes, a confident wrong answer produces real-world harm. The UI must communicate confidence levels, flag uncertainty, and have recovery paths. This belongs in V1.
The "Ask an Expert" tab is doing unexamined strategic work. The tab label implies AI isn't expert-level. But handled correctly, the expert tab is the trust anchor — knowing a human is one click away makes AI advice feel safer. Position it as escalation ("Get Expert Review"), not competition.
The /chat redirect needs state migration. Existing users have bookmarks, session state, and muscle memory. A hard redirect without handling produces support load at launch.

The Recommendation

Ship the morph, but redesign what the hero says and what the first AI response shows.

The architecture is close enough to correct. But the hero headline must answer the ChatGPT question before anyone types. Not "ask anything about your home" — something that names the irreplaceable output: real prices, code compliance, jurisdiction-aware permits.

Then the first AI response — whatever the query — must contain something structurally impossible to get from ChatGPT: a price-linked materials item, a code reference with jurisdiction, a local permit note. One concrete artifact that proves the system is different.

On the intent router: Add a visible micro-signal after routing — a line like "Got it — this sounds like a mid-project structural issue" — so users can confirm or redirect. Converts a silent failure point into a conversational checkpoint.

On "Ask an Expert": Rename to "Get Expert Review" or "Talk to a Pro" — implies the AI handled it and the human validates, which is the correct mental model.

On trust: Define confidence tiers before launch. High confidence answers render differently from low confidence ones. Users need a visual signal, not a disclaimer.

The One Thing to Do First

Write the hero headline — not as a design task, but as a strategic forcing function. The constraint: it must be something that is false or impossible for ChatGPT to say. If you can't write that headline, the design is not ready to build. If you can, everything downstream — morph trigger, suggestion chips, first AI response format, tab naming — falls into alignment around that answer.

The Contrarian — Full Response
The intent router is the load-bearing wall you haven't stress-tested. Everything downstream depends on it correctly classifying "my deck is rotting near the ledger board" as mid_project or full_project — but ambiguous messages will misclassify constantly, and the user has no way to correct it without breaking the conversational flow you're protecting. What's the recovery path when the AI treats a $15k structural repair like a quick question?

The "Plan This Project" CTA is a commitment trap. You're asking users to trust a pipeline they've never seen produce anything, based on one exchange. If the agent pipeline runs for 30 seconds and returns a mediocre plan, you've lost them permanently — and they'll tell ChatGPT about it.

The hero-to-morph pattern hides your differentiators at exactly the wrong moment. Suggestion chips and badges disappear the instant someone types. But that's when they most need to understand why this isn't just another chat box. Header drawers for power features assumes users will discover them. They won't. Tool inventory and shopping lists are core value props — burying them in drawers means most users will never know they exist.

The biggest unasked question: what happens to existing users with the /chat redirect?
The First Principles Thinker — Full Response
The real problem isn't the UI pattern. It's the value discovery problem. Users land, see a chat box, and their mental model immediately collapses to "this is just another ChatGPT wrapper." The Hero-to-Chat Morph doesn't solve this — it reinforces it. You're leading with the commodity feature (chat) and burying the differentiators as supporting text nobody reads.

Ask the foundational question: why do people bounce to ChatGPT? Not because the UI is wrong. Because they don't believe this will be better than what they already have. The morph pattern solves a navigation problem nobody has.

The actual job-to-be-done for a DIY homeowner isn't "chat with AI." It's "confidently complete a project without expensive mistakes." What's the fastest path to demonstrating irreplaceable value? Probably not a blank chat input — it's showing a concrete output they couldn't get from ChatGPT: a real materials list with Home Depot prices, a permit-aware project plan, a code compliance note. Let them see the output before they invest in the input.

The question to answer before any design decision: What is the single thing a user could see in 8 seconds that ChatGPT provably cannot give them? Design backwards from that.
The Expansionist — Full Response
The Hero-to-Morph is the right call, but you're underselling the biggest unlock here: this is the first DIY AI that remembers your house. Every generic AI session starts cold. You're sitting on Supabase — which means you can accumulate a persistent home profile across sessions. After someone asks about their deck, their bathroom tile, their HVAC — the app knows their house better than they do. That's not a chat feature, that's a home brain.

The agent pipeline for full_project is your actual moat, not the chat interface. ChatGPT can answer questions. It cannot produce a code-compliant materials list with real store prices and a phased timeline in one flow. Lead with the output, not the input. Show a finished project report in the hero — collapsed, scannable — before users even type anything.

Adjacent opportunity: contractor handoff. When a DIYer realizes mid-project they're in over their head, they need a contractor. The expert marketplace is positioned as Q&A, but it could be a warm handoff with full project context pre-loaded.

The suggestion chips aren't just UX affordances — they're the fastest way to communicate capability scope. Make them surprising: "Is my electrical panel safe for this?" signals code-compliance.
The Outsider — Full Response
The "Hero-to-Chat Morph" is clever to you because you've been staring at it. To a first-time visitor, the page just... changes. That's disorienting without a clear signal of why it changed.

The two-tab label problem is real: "Ask Anything" versus "Ask an Expert" — the natural question is "what's the first tab, a non-expert?" The implication is the AI is less authoritative. You want the opposite impression.

"What are you working on?" is fine as a prompt, but nothing on the hero immediately answers the question a skeptical new user is actually asking: "Why shouldn't I just use ChatGPT for this?" Your differentiator badges are doing that work, but badges get ignored. That answer needs to be in the headline or the first sentence below it.

The intent router is invisible by design, but invisibility has a cost. When the AI silently decides someone asking about a leaky faucet is a "quick question" versus a "full project," users who expected more will think the app is limited. You need a micro-signal that tells users the system understood what they're dealing with.

Suggestion chips: if they're generic, they signal nothing unique. They need to demonstrate capability, not just topics.
The Executor — Full Response
The intent router already exists at /lib/intelligence/intent-router.ts and the four classifications are already live in /app/api/chat/route.ts. LandingQuickChat is 427 lines and already handles streaming, materials extraction, and save-to-project.

Execution sequence: Week 1, two concrete tasks. First: add a redirect to /app/chat/page.tsx — one line, done in an hour. Second: strip the three-tab system from LandingHero.tsx and replace with two tabs.

Week 2, the hero morph. LandingQuickChat already tracks hasConversation = messages.length > 0. Add headline + suggestion chips that render when !hasConversation and hide when the first message fires. No new state needed.

Week 3, the CTA for full_project. The /api/chat/route.ts already calls classifyIntent(). When it returns full_project, emit a new SSE event type to the client. LandingQuickChat reads it and renders a "Plan This Project" button after the first AI response.

The GuidedBot replacement is the heaviest lift — roughly 2 weeks. Don't start there. Validate the morph and CTA pattern first. Header drawers can run in parallel.
Peer Review Highlights
Unanimous strongest response: First Principles Thinker (B) — "the 8-second ChatGPT test" was recognized by all 5 reviewers as the most actionable strategic framing.

Unanimous biggest blind spot: The Executor (E) — all reviewers flagged the purely tactical approach that assumes the design is correct before validating it.

Three themes emerged from "what all missed":
1. Trust failure / error states — confident wrong answers about structural or electrical work have real-world consequences (3/5 reviewers)
2. The "Ask an Expert" tab relationship to the AI tab — business model tension, trust anchor potential, cannibalization risk (2/5 reviewers)
3. Trust calibration signals — how the UI communicates confidence in AI answers (2/5 reviewers)
